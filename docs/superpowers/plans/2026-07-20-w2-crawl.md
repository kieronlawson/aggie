# W2 Firecrawl Change-Tracking Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Weekly Firecrawl change-tracking crawl of registry `crawl` sources, diff-based classification through P, and quiet-source reporting in the digest footer.

**Architecture:** New Firecrawl batch client functions; a pure `crawlRawItem` mapper; a deterministic self-page branch in P's layer-2; the `crawl` CLI orchestrating load → batch → poll → filter → process; footer wiring in the report path. See `docs/superpowers/specs/2026-07-20-w2-crawl-design.md`.

**Tech Stack:** TypeScript (Node 22 ESM via tsx), Ramda, vitest, Firecrawl v2 REST.

## Global Constraints

- **spoke-fp (CI-enforced):** all-`const`, no classes/loops/`if-else`/`switch` (single-if early return + single-level ternary OK), arrow functions with explicit return types, double quotes, semicolons, no trailing commas, ≤120-char lines, enums over string literals, ≤4 params (use options objects), **at most one throw and one try/catch per function**, no magic numbers (named constants). Recursion replaces loops (see `hardSplit` in mrkdwn.ts for precedent).
- **Imports:** `#src/*` subpaths only.
- **Verification:** `npm run check` before every commit. Workflows: `actionlint` + `zizmor` clean.
- **TurboPuffer gotcha:** never add an attribute to a query's `include_attributes`/filters unless it exists in the namespace schema. `source`, `url`, `story_id` have existed on item rows since phase 1 — safe. 
- **No logic in workflow YAML.**
- **Slack:** all output to `#intel-staging` (SlackChannel.IntelStaging) until the phase gate.

```typescript
enum ChangeStatus {
  New = "new",
  Same = "same",
  Changed = "changed",
  Removed = "removed"
}

const CHANGE_STATUSES: string[] = Object.values(ChangeStatus);

type CrawlPageResult = {
  url: string;
  title: string;
  markdown: string;
  changeStatus: ChangeStatus;
  diffText: string;
};

type BatchPageData = {
  markdown?: string;
  metadata?: { sourceURL?: string; title?: string };
  changeTracking?: { changeStatus?: string; diff?: { text?: string } | null };
};

type BatchStatusPayload = {
  status?: string;
  next?: string | null;
  data?: BatchPageData[];
};

type BatchResults = {
  status: string;
  pages: CrawlPageResult[];
};

const BATCH_STATUS_COMPLETED = "completed";
const BATCH_STATUS_FAILED = "failed";

const authHeaders = (): Record<string, string> => ({
  Authorization: `Bearer ${requireEnv("FIRECRAWL_API_KEY")}`,
  "Content-Type": "application/json"
});

const startChangeTrackingBatch = async (urls: string[]): Promise<string> => {
  const response = await fetch(`${FIRECRAWL_API_BASE}/batch/scrape`, {
    method: "POST",
    headers: authHeaders(),
    body: JSON.stringify({
      urls,
      formats: ["markdown", { type: "changeTracking", modes: ["git-diff"] }],
      proxy: "auto"
    })
  });
  if (!response.ok) {
    throw new Error(`Firecrawl batch start failed: HTTP ${String(response.status)} ${await response.text()}`);
  }
  const payload = (await response.json()) as { id?: string };
  const id = payload.id ?? "";
  if (id.length === 0) {
    throw new Error("Firecrawl batch start returned no job id");
  }
  return id;
};
```

```typescript
const toPageResult = (entry: BatchPageData): CrawlPageResult => {
  const status = entry.changeTracking?.changeStatus ?? "";
  return {
    url: entry.metadata?.sourceURL ?? "",
    title: entry.metadata?.title ?? "",
    markdown: entry.markdown ?? "",
    changeStatus: CHANGE_STATUSES.includes(status) ? (status as ChangeStatus) : ChangeStatus.Same,
    diffText: entry.changeTracking?.diff?.text ?? ""
  };
};

const fetchBatchPage = async (url: string): Promise<BatchStatusPayload> => {
  const response = await fetch(url, { headers: authHeaders() });
  if (!response.ok) {
    throw new Error(`Firecrawl batch status failed: HTTP ${String(response.status)} ${await response.text()}`);
  }
  return (await response.json()) as BatchStatusPayload;
};

const followNext = async (next: string | null, acc: BatchPageData[]): Promise<BatchPageData[]> => {
  if (next === null || next.length === 0) {
    return acc;
  }
  const page = await fetchBatchPage(next);
  return followNext(page.next ?? null, [...acc, ...(page.data ?? [])]);
};

const getBatchResults = async (jobId: string): Promise<BatchResults> => {
  const first = await fetchBatchPage(`${FIRECRAWL_API_BASE}/batch/scrape/${jobId}`);
  const rest = await followNext(first.next ?? null, []);
  const entries = [...(first.data ?? []), ...rest];
  return { status: first.status ?? "", pages: entries.map(toPageResult) };
};
```

Refactor the existing `remainingCredits`/`scrapeRaw` to use `authHeaders()` where it drops repetition. Export line adds: `BATCH_STATUS_COMPLETED, BATCH_STATUS_FAILED, type BatchResults, ChangeStatus, type CrawlPageResult, getBatchResults, startChangeTrackingBatch`.

```typescript
const layer2StoryId = async (
  item: RawItem,
  classified: ClassifyResult,
  nearest: TpufResultRow | undefined
): Promise<string | null> => {
  const similarity = nearest?.$dist === undefined ? 0 : 1 - nearest.$dist;
  if (nearest === undefined || similarity < DEDUPE_SIMILARITY_THRESHOLD) {
    return "";
  }
  if (str(nearest, "url") === item.url) {
    const selfStory = str(nearest, "story_id");
    return selfStory.length > 0 ? selfStory : String(nearest.id);
  }
  const verdict = await arbitrate(classified, nearest);
  if (verdict === DedupeVerdict.Duplicate) {
    return null;
  }
  if (verdict === DedupeVerdict.SameStory) {
    const neighbourStory = str(nearest, "story_id");
    return neighbourStory.length > 0 ? neighbourStory : String(nearest.id);
  }
  return "";
};
```

```typescript
import { ChangeStatus, type CrawlPageResult } from "#src/clients/firecrawl.ts";
import { type RawItem } from "#src/pipeline/types.ts";
import { type Relationship, type SourceRecord } from "#src/registry/types.ts";

const CONTEXT_EXCERPT_CHARS = 4000;
const NEW_PAGE_CHARS = 20000;

type CrawlItemOpts = {
  source: SourceRecord;
  relationship: Relationship;
  page: CrawlPageResult;
  nowIso: string;
};

const changedContent = (page: CrawlPageResult): string =>
  [
    "This tracked page changed since the last weekly crawl. Diff (git style):",
    page.diffText,
    "",
    "Current page context (excerpt):",
    page.markdown.slice(0, CONTEXT_EXCERPT_CHARS)
  ].join("\n");

/** Maps a crawl result to a RawItem for P; null = nothing to process (same/removed). */
const crawlRawItem = (opts: CrawlItemOpts): RawItem | null => {
  const { source, relationship, page, nowIso } = opts;
  const skippable = page.changeStatus === ChangeStatus.Same || page.changeStatus === ChangeStatus.Removed;
  if (skippable) {
    return null;
  }
  const changed = page.changeStatus === ChangeStatus.Changed;
  const pageTitle = page.title.length > 0 ? page.title : source.name;
  return {
    url: page.url,
    title: changed ? `${source.name} — page updated` : pageTitle,
    content: changed ? changedContent(page) : page.markdown.slice(0, NEW_PAGE_CHARS),
    published_at: nowIso,
    source: source.name,
    vertical: source.vertical,
    competitor: source.competitor,
    relationship
  };
};

export { CONTEXT_EXCERPT_CHARS, crawlRawItem, type CrawlItemOpts, NEW_PAGE_CHARS };
```

- [ ] **Step 1: Implement `src/cli/crawl.ts`**

Behavior (follow ingest.ts idioms; constants named, no magic numbers):

1. `loadActiveSources(SourceKind.Crawl)`. Empty → log + post "📭 no active crawl sources" note, return.
2. Credits guard: `remainingCredits()`; if below `sources.length`, throw `new Error` naming the numbers (caught by the top-level handler → ❌ post).
3. `startChangeTrackingBatch(urls)` → poll loop: recursive `pollUntilDone(jobId, deadlineMs)` — `getBatchResults`; `completed` → return results; `failed` OR past deadline → single throw with a ternary message; otherwise `await sleep(POLL_INTERVAL_MS)` and recurse. `POLL_INTERVAL_MS = 15_000`, `POLL_TIMEOUT_MS = 900_000`, `sleep` = `new Promise((resolve) => setTimeout(resolve, ms))`.
4. Map competitor → relationship: from `loadCompetitors()` build a name→relationship record; relationship = source.competitor non-empty ? map lookup (fallback `Relationship.Displace`) : `Relationship.Regulatory`.
5. Match each result page to its source by URL (`R.indexBy(R.prop("url"), sources)`); pages with no matching source are counted and skipped (log them).
6. Build items via `crawlRawItem`; drop nulls.
7. Seen filter for NEW pages only (changed pages always process): for new-status items, query the vertical's items namespace `["url", "In", urls]` (see `seenUrls` in ingest.ts for the pattern) and drop matches.
8. `sequentially(items, processRawItem)` with per-item try/catch collecting failures as `"<source name>: <error message>"` (one try/catch in one helper function).
9. Post summary to staging: `🕸️ Aggie crawl: N pages checked — X changed, Y new (Z of them already seen), W unchanged/removed; stored S, merged M. Failures: ...` (⚠️ lines only when failures exist). Log the same to console.
